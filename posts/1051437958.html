执行镜像 生成容器

  • -d: 后台运行容器，并返回容器ID；
  • -i: 以交互模式运行容器，通常与 -t 同时使用；
  • -t: 为容器重新分配一个伪输入终端，通常与 -i 同时使用；

根据当前位置的dockerfile构建镜像

  • docker build -t 镜像名称 文件位置
  • –tag, -t: 镜像的名字及标签，通常 name:tag 或者 name 格式；
